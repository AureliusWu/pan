# AGENTS.md

## 项目识别

- 目录名：`pan`
- 中文名：盘中宝
- 当用户说「盘中宝」时，指的就是本项目：`pan`
- 线上形态：GitHub Pages 托管的手机端基金盘中估值 PWA

## 项目定位

盘中宝是独立维护的移动端基金盘中观察工具，专注于基金盘中估值、指数/黄金行情、持仓市值、关注模式和 GitHub Gist 多设备同步。它不是蜉蝣基金测试版，也不是司南基金的子模块。

## 技术与结构

- 技术栈：原生 HTML/CSS/JavaScript，无构建工具、无 npm 依赖。
- 入口文件：`index.html`
- 业务逻辑：`js/app.js`
- 样式：`css/style.css`
- PWA：`manifest.json`、`sw.js`
- 视觉方向：青绿插金主题，移动端优先。
- 部署：推送 `main` 后由 GitHub Pages 发布。

## 数据源与关键约定

- 基金估值：天天基金 JSONP + 东方财富备用数据。
- 指数行情：腾讯行情 JSONP。
- 黄金行情：独立行情源与缓存。
- 云同步：GitHub Gist API。
- 本地存储 key 使用 `panzhongbao_` 前缀，例如 `panzhongbao_holdings_v1`，不要与蜉蝣基金的 `fuyu_` 前缀混用。
- 基金详情和持仓数据通过脚本注入/公开接口获取，注意 JSONP 全局回调和并发请求隔离。

## 开发规则

- 保持零依赖纯前端，不引入框架或构建流程。
- 任何持仓、Gist 同步、删除 tombstone、导入导出相关改动，都要兼容已有 localStorage 数据。
- 修改 UI 时保持手机端首屏优先，不要把布局改成桌面优先。
- 涉及通知、自动刷新、下拉刷新时，要检查休市、午休、周末和网络失败状态。
- 不在代码或文档中写真实 GitHub Token。

## 项目边界

- 不要把本项目与 `FundVal`（蜉蝣基金）混淆：两者都是基金盘中工具，但盘中宝使用 `panzhongbao_` 存储前缀和青绿插金主题。
- 不要把本项目与 `fund-compass`（司南基金）混淆：司南基金负责选基、择时、回测和资产分析。
